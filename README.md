# TGC SCRATCH Workspace Engine

A minimalist, high-performance, multithreaded command-line automation toolkit designed to manage Scratch account clusters and interactions using the `scratchattach` wrapper.

---

## 🔥 Key Technical Capabilities

* **Zero-Setup Initialization:** Automatically scans, installs, or upgrades missing system libraries (`scratchattach`) immediately upon launch.
* **Asynchronous Execution Arrays:** Spawns isolation threads for worker interactions. Follow, unfollow, and relationship audit checks happen concurrently.
* **Unified Memory Matrix:** Keeps a localized config file (`config.txt`) storing the global access password alongside your customized worker registry across engine runs.
* **Fail-Safe Log Overflow Protection:** An integrated log handling system prevents display corruption by keeping trace strings capped and dynamic interface bounds strict.

## 🚀 Speed-Up Guide

### Prerequisites
Make sure Python is installed on your computer. You do not need to install any packages manually.

### Initializing the Script
Run the file from your local console wrapper:
```bash
python lazy_scratch.py
```
*Note: On your very first run, the tool will instantly run a pip-dependency validation shield, prepare your configuration files, and launch with `abc123` as the default global master password.*

## ⌨️ Dashboard Controller Index

Once inside the engine loop, type one of the following lowercase action switches and press `Enter`:


| Key Command | Internal Automation Trigger | Expected Operation Details |
| :---: | :--- | :--- |
| **`f`** | **Bind target profile** | Declares the global profile coordinate target handle for all mass scripts. |
| **`p`** | **Update worker password** | Overwrites the local global worker pool login token saved inside storage memory. |
| **`n`** | **Add worker account** | Appends one or more handles (comma-separated format) directly into database registry. |
| **`r`** | **Remove worker account** | Detaches and cleans specific accounts from the pool file array. |
| **`g`** | **Mass follow target** | Deploys active thread batches ordering every worker account to follow your set target. |
| **`c`** | **Mass unfollow target** | Deploys active thread batches to mass-unfollow the designated profile coordinate. |
| **`i`** | **Check follow status** | Queries relationship endpoints to see which accounts are following the target. |
| **`s`** | **Inspect target stats** | Pulls real-time public network metrics, including current follower tracking counts. |
| **`l`** | **View accounts database** | Spits out a list stream displaying every single profile stored inside your active pool. |
| **`q`** | **Quit automation engine** | Safely breaks thread processes, flashes termination stats, and resets the terminal. |
